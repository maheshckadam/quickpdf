Why I built Quickpdf — a free, private alternative to bloated PDF sites

April 1, 2026 · 4 min read

I needed to merge three PDFs. I went to iLovePDF, did the merge, and started downloading. Then a popup appeared offering me Premium for $7.99/month. I clicked away. Another popup asked me to sign up. I clicked away again. Eventually the file downloaded.

I closed the tab annoyed, and thought: this isn't a hard task. Combining PDFs is a 50-line JavaScript program. Why is the experience like buying a timeshare?

The state of online PDF tools

Most "free PDF tool" websites follow a familiar pattern:

  • Upload your file to their server (where it sits on disk for some retention window)
  • Show you ads, popups, and tracking pixels while it processes
  • Daily limits on usage to push you toward a paid tier
  • Watermarks on output unless you upgrade
  • Email collection in exchange for "extra" features

None of this is necessary for the task at hand. Browsers can do PDF manipulation locally. They have for years. The "send your file to a server" model is a business choice, not a technical requirement.

What changed

The JavaScript ecosystem got really good. Libraries like pdf-lib (PDF reading and writing), pdf.js (PDF rendering, by Mozilla), Tesseract.js (OCR), and Transformers.js (in-browser AI) make it possible to build a full suite of PDF tools that never need a server. Static HTML and JavaScript can do the work that used to require a backend with file storage and processing queues.

This isn't speculative — every major PDF feature (merge, split, compress, watermark, sign, OCR, AI summarization) now has a robust JS implementation.

So I built Quickpdf

Over a few weekends, I built and shipped Quickpdf. The goals were simple:

  • Free forever, no tiers, no upgrade prompts
  • No signup, no email collection
  • No file uploads — everything in the browser
  • No watermarks on output
  • No tracking, no ads
  • No file size limits beyond what your browser can handle

The whole site is static HTML, CSS, and JS. It's hosted on Vercel's free hobby tier. Bandwidth costs me roughly nothing because Vercel's CDN absorbs it.

"How is this sustainable?"

Honestly: it doesn't need to be sustainable in the traditional sense. The marginal cost of one more user using a static site on a CDN is essentially zero. There are no servers to scale, no databases to back up, no workers to spin up. If a million people use Quickpdf tomorrow, my hosting bill goes up by maybe $20.

That said, if it grows beyond what the free tier supports, I'll figure out a way to keep it free — donations, sponsorships, optional paid extras for power users. The core stays free regardless.

What's next

I want Quickpdf to be the obvious choice when you need to do something with a PDF. Not the most feature-packed (it isn't), not the prettiest (Adobe still wins there), but the most respectful of your time and privacy.

Currently shipping: Merge, Split, Compress, Organize, Watermark, Sign, Unlock, Protect, OCR, AI Summarizer, PDF↔Word, PDF↔JPG, Edit. Soon: Image to PDF, Compare PDFs, Rotate (standalone).

If you have ideas, complaints, or bug reports, I'd love to hear them. Quickpdf gets better when people use it and tell me what's missing.
